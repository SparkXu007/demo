# 数字政通 Java 岗位面试真题

## 一、Java基础核心考点

### 1.1 面向对象特性
- 请详细解释封装、继承、多态的概念，并结合实际代码举例说明多态的实现场景。
  **答案：** 
  - **封装**：将数据和操作数据的方法绑定在一起，隐藏内部实现细节，通过公共接口访问。例如：private 属性 + public getter/setter 方法。
  - **继承**：子类继承父类的属性和方法，实现代码复用。使用 extends 关键字。
  - **多态**：同一行为具有多种不同表现形式。分为编译时多态（方法重载）和运行时多态（方法重写）。
  
  **多态实现示例**：
  ```java
  class Animal {
      void makeSound() { System.out.println("动物发声"); }
  }
  class Dog extends Animal {
      @Override
      void makeSound() { System.out.println("汪汪汪"); }
  }
  class Cat extends Animal {
      @Override
      void makeSound() { System.out.println("喵喵喵"); }
  }
  // 使用时：Animal animal = new Dog(); animal.makeSound(); // 输出"汪汪汪"
  ```

- 里氏替换原则的具体内容是什么？在继承关系中如何避免违反该原则？
  **答案：**
  - **内容**：子类对象必须能够替换掉所有父类对象，且程序行为不变。即子类应该完全支持父类的功能，不能改变父类原有的语义和行为。
  - **避免违反**：
    1. 子类不要覆盖父类的非抽象方法
    2. 子类可以增加自己的特有方法
    3. 当子类需要实现父类的抽象方法时，不要强化前置条件或弱化后置条件
    4. 如果父类有具体方法，尽量使用组合而非继承

- 组合与继承的区别是什么？在什么场景下更适合使用组合而非继承？
  **答案：**
  - **区别**：
    - 继承是"is-a"关系，表示类之间的层次关系，编译时静态绑定
    - 组合是"has-a"关系，表示整体与部分的关系，运行时动态组装
  - **优先使用组合的场景**：
    1. 子类不完全是父类的特殊类型
    2. 需要复用多个类的功能（Java 单继承限制）
    3. 避免继承层次过深
    4. 需要灵活替换组件
    5. 遵循"合成复用原则"，降低耦合度

### 1.2 集合框架
- HashMap 的底层实现原理是什么？JDK 1.8 对 HashMap 做了哪些优化？^
  **答案：**
  - **底层实现**：数组 + 链表 + 红黑树（JDK1.8）。通过 hash 值计算索引位置，解决哈希冲突使用链地址法。
  - **JDK1.8 优化**：
    1. 链表长度超过 8 且数组长度超过 64 时，链表转为红黑树，提高查询效率（O(n)→O(logn)）
    2. 链表长度小于 6 时，红黑树转回链表
    3. 插入方式从头插法改为尾插法，避免扩容时链表成环
    4. 扩容机制优化，重新计算 hash 更均匀

- HashMap、ConcurrentHashMap、Hashtable 之间的区别有哪些？分别适用于什么场景？^
  **答案：**
  | 特性 | HashMap | ConcurrentHashMap | Hashtable |
  |------|---------|-------------------|----------|
  | 线程安全 | 否 | 是（CAS+synchronized） | 是（全表锁） |
  | null 键/值 | 允许 | 不允许 | 不允许 |
  | 性能 | 高 | 高（分段锁/CAS） | 低 |
  | 迭代器 | 快失败 | 弱一致性 | 快失败 |
  - **适用场景**：
    - HashMap：单线程环境
    - ConcurrentHashMap：多线程并发读写
    - Hashtable：已淘汰，不建议使用

- ArrayList 和 LinkedList 的底层数据结构是什么？它们在增删改查操作上的性能差异如何？
  **答案：**
  - **底层结构**：
    - ArrayList：动态数组
    - LinkedList：双向链表
  - **性能差异**：
    | 操作 | ArrayList | LinkedList |
    |------|-----------|------------|
    | 随机访问 | O(1) | O(n) |
    | 头部插入/删除 | O(n) | O(1) |
    | 尾部插入/删除 | O(1)  amortized | O(1) |
    | 中间插入/删除 | O(n) | O(1)（定位后） |
  - **选择建议**：频繁查询用 ArrayList，频繁插入删除用 LinkedList

### 1.3 异常处理
- Java 中的异常体系是怎样的？Checked Exception 和 Unchecked Exception 有什么区别？
  **答案：**
  - **异常体系**：
    ```
    Throwable
    ├── Error（系统错误，如 OutOfMemoryError）
    └── Exception
        ├── Checked Exception（受检异常，如 IOException）
        └── Unchecked Exception（运行时异常，如 NullPointerException）
    ```
  - **区别**：
    | Checked Exception | Unchecked Exception |
    |-------------------|--------------------|
    | 编译期强制处理 | 编译期不强制处理 |
    | 可预见的异常 | 编程错误导致 |
    | 必须 try-catch 或 throws | 可选择性处理 |
    | 如：IOException、SQLException | 如：NullPointerException、ArrayIndexOutOfBoundsException |

- 请说明 try-catch-finally 块的执行顺序，当 try 块中存在 return 语句时，finally 块是否会执行？
  **答案：**
  - **执行顺序**：try → catch（如果有异常）→ finally → return
  - **finally 一定会执行**，除非：
    1. 调用 System.exit()
    2. 线程被杀死
    3. JVM 崩溃
  - **注意**：如果 finally 中有 return，会覆盖 try/catch 中的 return 值
  ```java
  int test() {
      try { return 1; }
      finally { return 2; } // 最终返回 2
  }
  ```

### 1.4 其他基础问题
- Java 的值传递和引用传递有什么区别？为什么说 Java 中只有值传递？
  **答案：**
  - **值传递**：传递的是参数的副本，修改不影响原值
  - **引用传递**：传递的是参数的地址，修改会影响原值
  - **Java 只有值传递**：
    1. 基本类型：传递值的副本
    2. 引用类型：传递引用的副本（对象地址的拷贝），但引用本身是按值传递
    ```java
    void change(int[] arr) {
        arr[0] = 100; // 修改数组内容，原数组会变
        arr = new int[5]; // 修改引用指向，原引用不变
    }
    ```

- 请解释 JDK、JRE、JVM 之间的关系，以及 Java 跨平台特性的实现原理。^
  **答案：**
  - **关系**：
    - **JDK**（Java Development Kit）：开发工具包，包含 JRE + 开发工具（javac、java 等）
    - **JRE**（Java Runtime Environment）：运行环境，包含 JVM + 核心类库
    - **JVM**（Java Virtual Machine）：Java 虚拟机，负责执行字节码
    - 包含关系：JDK ⊃ JRE ⊃ JVM
  - **跨平台原理**：
    1. Java 源代码编译成字节码（.class 文件）
    2. 字节码不针对特定处理器，而是面向 JVM
    3. 不同平台安装对应的 JVM 实现
    4. JVM 将字节码翻译成特定平台的机器码
    5. "Write Once, Run Anywhere"

## 二、JVM 与性能调优

### 2.1 内存模型与区域划分
- JVM 运行时数据区由哪些部分组成？每个部分的作用是什么？JDK 8 之后永久代被替换为元空间，带来了哪些变化？^
  **答案：**
  - **组成部分**：
    1. **程序计数器**：记录当前线程执行的字节码行号，线程私有
    2. **Java 虚拟机栈**：存储栈帧（局部变量表、操作数栈等），线程私有
    3. **本地方法栈**：为 Native 方法服务，线程私有
    4. **堆**：存储对象实例，GC 主要区域，线程共享
    5. **方法区**：存储类信息、常量、静态变量，线程共享
  - **JDK8 变化**：
    - 永久代 → 元空间（Metaspace）
    - 元空间使用本地内存，不再受 JVM 堆大小限制
    - 减少 Full GC 频率，避免 PermGen OOM

- 请描述对象在堆内存中的分配过程，以及新生代和老年代的垃圾回收策略有何不同。^
  **答案：**
  - **分配过程**：
    1. 优先在 Eden 区分配
    2. Eden 满时触发 Minor GC，存活对象移到 Survivor 区
    3. 每次 GC 后年龄 +1，达到阈值（默认 15）进入老年代
    4. 大对象直接进入老年代
  - **回收策略差异**：
    | 新生代 | 老年代 |
    |--------|--------|
    | Minor GC | Major GC/Full GC |
    | 复制算法 | 标记 - 清除/标记 - 整理 |
    | 频繁、快速 | 低频率、耗时长 |
    | 对象朝生夕死 | 对象存活率高 |

### 2.2 垃圾回收机制
- 常见的垃圾回收算法有哪些？分别适用于什么场景？^
  **答案：**
  1. **标记 - 清除算法**：标记存活对象，清除未标记对象。会产生碎片，适合老年代
  2. **标记 - 复制算法**：将存活对象复制到另一块内存。无碎片，但浪费空间，适合新生代
  3. **标记 - 整理算法**：标记后将存活对象向一端移动。无碎片且不浪费空间，适合老年代
  4. **分代收集算法**：根据对象生命周期分区回收，现代 JVM 采用

- CMS 和 G1 垃圾回收器的区别是什么？G1 是如何实现低停顿的？^
  **答案：**
  - **CMS**（Concurrent Mark Sweep）：
    - 基于标记 - 清除，老年代回收器
    - 并发标记，减少停顿，但会产生碎片
    - 对 CPU 敏感，可能发生浮动失败
  - **G1**（Garbage First）：
    - 基于 Region 分区，可管理整个堆
    - 可预测停顿时间模型
    - 无内存碎片，使用整理算法
  - **G1 低停顿关键**：
    1. 分区管理，只回收价值最大的 Region
    2. 并发标记和整理
    3. 停顿预测模型，控制 GC 时间在目标范围内

- 如何排查 JVM 内存泄漏问题？常用的排查工具和命令有哪些？^
  **答案：**
  - **排查步骤**：
    1. 监控内存使用情况（jstat、VisualVM）
    2. 生成堆转储文件（jmap -dump）
    3. 分析 dump 文件（MAT、JProfiler）
    4. 定位泄漏对象和引用链
    5. 修复代码（常见原因：静态集合、未关闭资源、监听器等）
  - **常用工具**：
    - jps：查看 Java 进程
    - jstat：监控 GC 和内存
    - jmap：生成堆 dump
    - jhat：分析 heap dump
    - VisualVM、MAT、JProfiler：图形化分析工具

### 2.3 类加载机制
- 请简述类加载的双亲委派模型，以及它的优点是什么？
  **答案：**
  - **双亲委派模型**：
    1. 类加载器收到请求时，先委托给父加载器
    2. 逐级向上，直到启动类加载器
    3. 父加载器无法加载时，子加载器才尝试加载
  - **层次结构**：
    Bootstrap ClassLoader → Extension ClassLoader → Application ClassLoader → Custom ClassLoader
  - **优点**：
    1. 保证核心类库安全（防止自定义 String 等类）
    2. 避免类的重复加载
    3. 维持 Java 类型体系一致性

- 自定义类加载器需要继承哪个类？在什么场景下需要自定义类加载器？
  **答案：**
  - **继承**：ClassLoader 类，重写 findClass() 方法
  - **应用场景**：
    1. 加载非标准路径的类（如网络、数据库）
    2. 实现热部署（如 Tomcat）
    3. 实现模块化、插件化架构
    4. 加密解密类文件
    5. 隔离不同版本的同一类（如 SPI 机制）

## 三、多线程与并发编程

### 3.1 线程基础
- Java 中创建线程的方式有哪些？请分别给出代码示例。^
  **答案：**
  1. **继承 Thread 类**：
     ```java
     class MyThread extends Thread {
         public void run() { System.out.println("线程运行"); }
     }
     new MyThread().start();
     ```
  2. **实现 Runnable 接口**：
     ```java
     class MyRunnable implements Runnable {
         public void run() { System.out.println("线程运行"); }
     }
     new Thread(new MyRunnable()).start();
     ```
  3. **实现 Callable 接口**（有返回值）：
     ```java
     class MyCallable implements Callable<String> {
         public String call() { return "结果"; }
     }
     FutureTask<String> task = new FutureTask<>(new MyCallable());
     new Thread(task).start();
     ```
  4. **使用线程池**：
     ```java
     ExecutorService executor = Executors.newFixedThreadPool(10);
     executor.execute(() -> System.out.println("线程池任务"));
     ```

- 线程的生命周期包含哪些状态？状态之间是如何转换的？^
  **答案：**
  - **6 种状态**：NEW、RUNNABLE、BLOCKED、WAITING、TIMED_WAITING、TERMINATED
  - **状态转换**：
    1. NEW → RUNNABLE：调用 start()
    2. RUNNABLE ↔ BLOCKED：等待 synchronized 锁
    3. RUNNABLE ↔ WAITING：wait()/notify()
    4. RUNNABLE ↔ TIMED_WAITING：sleep()/wait(timeout)
    5. 任意状态 → TERMINATED：run() 结束或异常退出

- 线程的 start() 方法和 run() 方法有什么区别？
  **答案：**
  | start() | run() |
  |---------|-------|
  | 启动新线程 | 普通方法调用 |
  | JVM 调用 run() | 当前线程执行 |
  | 只能调用一次 | 可多次调用 |
  | 异步执行 | 同步执行 |

### 3.2 线程安全与同步
- synchronized 关键字的作用是什么？它的底层实现原理是怎样的？^
  **答案：**
  - **作用**：保证同一时刻只有一个线程执行同步代码块或方法
  - **底层原理**：
    1. 基于对象监视器（Monitor）
    2. 字节码中使用 monitorenter/monitorexit 指令
    3. JDK1.6 后优化：偏向锁 → 轻量级锁 → 重量级锁
    4. 锁升级机制减少性能开销
  - **使用方式**：
    ```java
    synchronized(this) { } // 同步代码块
    synchronized static void method() { } // 静态方法锁 Class
    synchronized void method() { } // 实例方法锁 this
    ```

- synchronized 和 ReentrantLock 的区别有哪些？分别适用于什么场景？^
  **答案：**
  | 特性 | synchronized | ReentrantLock |
  |------|--------------|---------------|
  | 实现层面 | JVM 层面 | API 层面 |
  | 锁释放 | 自动释放 | 手动 lock/unlock |
  | 可中断 | 否 | 是（lockInterruptibly） |
  | 超时获取 | 否 | 是（tryLock(timeout)） |
  | 公平锁 | 非公平 | 可选公平/非公平 |
  | 条件变量 | 单一（wait/notify） | 多个（Condition） |
  - **选择建议**：
    - synchronized：简单场景，代码简洁
    - ReentrantLock：需要高级功能（公平锁、可中断、超时等）

- volatile 关键字的作用是什么？它能保证线程安全吗？为什么？^
  **答案：**
  - **作用**：
    1. 保证可见性：修改立即刷新到主内存，其他线程立即可见
    2. 禁止指令重排序：通过内存屏障实现
  - **不能保证线程安全**：
    - 不保证原子性（如 i++ 操作）
    - 适合场景：状态标记、单例模式双重检查
  ```java
  volatile boolean flag = false;
  if (!flag) { // 可见性保证
      flag = true; // 但这里不是原子操作
  }
  ```

### 3.3 线程池
- Java 中常见的线程池类型有哪些？它们的适用场景分别是什么？^
  **答案：**
  1. **FixedThreadPool**：固定大小线程池，适合负载较重场景
  2. **CachedThreadPool**：缓存线程池，适合执行大量短期异步任务
  3. **SingleThreadExecutor**：单线程池，保证任务顺序执行
  4. **ScheduledThreadPool**：支持定时和周期性任务
  5. **WorkStealingPool**：工作窃取池，ForkJoinPool，适合 CPU 密集型
  - **注意**：建议使用 ThreadPoolExecutor 自定义参数

- ThreadPoolExecutor 的核心参数有哪些？请解释每个参数的含义。^
  **答案：**
  ```java
  new ThreadPoolExecutor(
      int corePoolSize,        // 核心线程数
      int maximumPoolSize,     // 最大线程数
      long keepAliveTime,      // 非核心线程空闲存活时间
      TimeUnit unit,           // 时间单位
      BlockingQueue<Runnable> workQueue, // 任务队列
      ThreadFactory threadFactory,       // 线程工厂
      RejectedExecutionHandler handler   // 拒绝策略
  )
  ```
  - **工作流程**：
    1. 任务提交，核心线程未满 → 创建核心线程执行
    2. 核心线程已满 → 加入队列
    3. 队列满 → 创建非核心线程
    4. 超过最大线程数 → 执行拒绝策略

- 线程池的拒绝策略有哪些？当任务队列满了之后，线程池会如何处理新的任务？^
  **答案：**
  - **4 种拒绝策略**：
    1. **AbortPolicy**（默认）：抛出 RejectedExecutionException
    2. **CallerRunsPolicy**：由调用线程执行该任务
    3. **DiscardPolicy**：直接丢弃任务
    4. **DiscardOldestPolicy**：丢弃队列中最老的任务，然后重试
  - **处理流程**：
    核心线程 → 队列 → 非核心线程 → 拒绝策略

## 四、数据库相关

### 4.1 MySQL 基础
- MySQL 的存储引擎有哪些？InnoDB 和 MyISAM 的区别是什么？^
  **答案：**
  - **常见引擎**：InnoDB、MyISAM、Memory、Archive
  - **InnoDB vs MyISAM**：
    | 特性 | InnoDB | MyISAM |
    |------|--------|--------|
    | 事务支持 | 支持（ACID） | 不支持 |
    | 锁粒度 | 行级锁 | 表级锁 |
    | 外键约束 | 支持 | 不支持 |
    | 崩溃恢复 | 支持 | 较差 |
    | COUNT(*) | 需全表扫描 | 内部计数器 |
    | 适用场景 | 高频写入、事务 | 读多写少 |
  - **默认引擎**：InnoDB（MySQL 5.5+）

- 请解释索引的作用，以及 B+ 树索引的底层实现原理。^
  **答案：**
  - **作用**：提高查询效率，降低时间复杂度（O(n)→O(logn)）
  - **B+ 树原理**：
    1. 非叶子节点只存索引，不存数据
    2. 叶子节点存储所有数据和指针
    3. 叶子节点形成有序链表
    4. 多路平衡查找树，减少磁盘 IO
  - **优势**：
    - 查询效率稳定
    - 范围查询友好
    - 支持排序

- 如何进行 SQL 语句优化？有哪些常用的优化策略？
  **答案：**
  - **优化步骤**：
    1. 使用 EXPLAIN 分析执行计划
    2. 查看 type、key、rows 等指标
    3. 定位慢查询（slow query log）
  - **优化策略**：
    1. **索引优化**：最左前缀原则、覆盖索引、避免索引失效
    2. **SQL 改写**：避免 SELECT *、用 UNION ALL 代替 UNION
    3. **结构优化**：分库分表、读写分离

### 4.2 MyBatis 框架
- MyBatis 的工作原理是什么？它与 JDBC 相比有哪些优势？^

- Redis 的持久化机制有哪些？RDB 和 AOF 的区别是什么？
  **答案：**
  - **RDB（Redis Database）**：
    - 原理：定时快照，保存数据到 dump.rdb
    - 优点：文件紧凑、恢复快、适合备份
    - 缺点：可能丢失最后一次快照后的数据
    - 触发：SAVE、BGSAVE、自动触发
  
  - **AOF（Append Only File）**：
    - 原理：记录每次写操作，重启时重放
    - 优点：数据更安全，最多丢失 1 秒
    - 缺点：文件较大、恢复慢
    - 重写：BGREWRITEAOF 压缩日志
  
  - **混合持久化**（Redis 4.0+）：
    - RDB 快照 + AOF 增量，兼顾两者优点
  
  - **选择建议**：
    - 只作缓存：关闭持久化
    - 数据重要：开启 AOF
    - 折中方案：RDB + AOF

- 如何使用 Redis 实现分布式锁？在实现过程中需要注意哪些问题？^
  **答案：**
  - **基本实现**：
    ```java
    // 加锁
    boolean lock = redisTemplate.opsForValue()
        .setIfAbsent("lock", "value", 10, TimeUnit.SECONDS);
    try {
        if (lock) {
            // 执行业务
        }
    } finally {
        // 释放锁（Lua 脚本保证原子性）
        String script = "if redis.call('get',KEYS[1]) == ARGV[1] then return redis.call('del',KEYS[1]) else return 0 end";
        redisTemplate.execute(new DefaultRedisScript<>(script, Long.class), 
                              Collections.singletonList("lock"), "value");
    }
    ```
  
  - **注意事项**：
    1. **互斥性**：使用 SETNX 或 setIfAbsent
    2. **超时时间**：防止死锁，设置过期时间
    3. **原子性**：加锁和设置过期时间要原子操作（Redis 2.8+ SET key value EX seconds NX）
    4. **锁误删**：验证 value 是否为当前线程（UUID 标识）
    5. **锁续期**：业务执行时间长时使用 WatchDog 机制（Redisson）
    6. **高可用**：RedLock 算法解决主从切换问题

- 请描述二叉树的前序、中序、后序遍历的递归和非递归实现方式。^
  **答案：**
  ```java
  class TreeNode {
      int val;
      TreeNode left, right;
      TreeNode(int x) { val = x; }
  }
  
  // 前序遍历：根→左→右
  // 递归
  void preorderRecursive(TreeNode root) {
      if (root == null) return;
      visit(root);
      preorderRecursive(root.left);
      preorderRecursive(root.right);
  }
  // 非递归（栈）
  void preorderIterative(TreeNode root) {
      Stack<TreeNode> stack = new Stack<>();
      while (root != null || !stack.isEmpty()) {
          while (root != null) {
              visit(root);
              stack.push(root);
              root = root.left;
          }
          root = stack.pop().right;
      }
  }
  
  // 中序遍历：左→根→右
  // 递归
  void inorderRecursive(TreeNode root) {
      if (root == null) return;
      inorderRecursive(root.left);
      visit(root);
      inorderRecursive(root.right);
  }
  // 非递归（栈）
  void inorderIterative(TreeNode root) {
      Stack<TreeNode> stack = new Stack<>();
      while (root != null || !stack.isEmpty()) {
          while (root != null) {
              stack.push(root);
              root = root.left;
          }
          root = stack.pop();
          visit(root);
          root = root.right;
      }
  }
  
  // 后序遍历：左→右→根
  // 递归
  void postorderRecursive(TreeNode root) {
      if (root == null) return;
      postorderRecursive(root.left);
      postorderRecursive(root.right);
      visit(root);
  }
  // 非递归（双栈）
  void postorderIterative(TreeNode root) {
      Stack<TreeNode> s1 = new Stack<>();
      Stack<TreeNode> s2 = new Stack<>();
      s1.push(root);
      while (!s1.isEmpty()) {
          TreeNode node = s1.pop();
          s2.push(node);
          if (node.left != null) s1.push(node.left);
          if (node.right != null) s1.push(node.right);
      }
      while (!s2.isEmpty()) {
          visit(s2.pop());
      }
  }
  ```

### 7.2 算法
- 请说明快速排序的实现原理和时间复杂度。
  **答案：**
  - **实现原理**：
    1. 选择基准值（pivot）
    2. 分区：小于 pivot 的放左边，大于的放右边
    3. 递归对左右子数组重复上述过程
  
  - **代码实现**：
    ```java
    void quickSort(int[] arr, int low, int high) {
        if (low < high) {
            int pi = partition(arr, low, high);
            quickSort(arr, low, pi - 1);
            quickSort(arr, pi + 1, high);
        }
    }
    
    int partition(int[] arr, int low, int high) {
        int pivot = arr[high];
        int i = low - 1;
        for (int j = low; j < high; j++) {
            if (arr[j] < pivot) {
                i++;
                swap(arr, i, j);
            }
        }
        swap(arr, i + 1, high);
        return i + 1;
    }
    ```
  
  - **时间复杂度**：
    - 最好：O(n log n)
    - 平均：O(n log n)
    - 最坏：O(n²)（已排序数组）
  - **空间复杂度**：O(log n)（递归栈）
  - **稳定性**：不稳定

- 如何实现链表的反转？请给出代码示例。
  **答案：**
  ```java
  class ListNode {
      int val;
      ListNode next;
      ListNode(int x) { val = x; }
  }
  
  // 迭代法
  ListNode reverseListIterative(ListNode head) {
      ListNode prev = null;
      ListNode curr = head;
      while (curr != null) {
          ListNode nextTemp = curr.next;
          curr.next = prev;
          prev = curr;
          curr = nextTemp;
      }
      return prev;
  }
  
  // 递归法
  ListNode reverseListRecursive(ListNode head) {
      if (head == null || head.next == null) return head;
      ListNode newHead = reverseListRecursive(head.next);
      head.next.next = head;
      head.next = null;
      return newHead;
  }
  ```

- 动态规划的核心思想是什么？请结合实际问题举例说明动态规划的应用。
  **答案：**
  - **核心思想**：
    1. 将复杂问题分解为重叠子问题
    2. 保存子问题的解，避免重复计算
    3. 通过状态转移方程递推求解
  
  - **关键要素**：
    1. 最优子结构
    2. 状态定义
    3. 状态转移方程
    4. 边界条件
  
  - **经典案例：爬楼梯**
    ```java
    // 问题：每次可爬 1 或 2 阶，爬到 n 阶有多少种方法？
    // 状态定义：dp[i] 表示到第 i 阶的方法数
    // 转移方程：dp[i] = dp[i-1] + dp[i-2]
    int climbStairs(int n) {
        if (n <= 2) return n;
        int[] dp = new int[n + 1];
        dp[1] = 1;
        dp[2] = 2;
        for (int i = 3; i <= n; i++) {
            dp[i] = dp[i - 1] + dp[i - 2];
        }
        return dp[n];
    }
    ```
  
  - **其他经典问题**：
    - 0-1 背包问题
    - 最长公共子序列
    - 编辑距离
    - 最大子序和

  | 流量控制 | 有（滑动窗口） | 无 |
  | 数据量 | 字节流，无边界 | 数据报，有边界 |
  | 头部开销 | 20-60 字节 | 8 字节 |
  
  - **适用场景**：
    - **TCP**：文件传输、邮件、网页浏览（HTTP/HTTPS）、数据库连接
    - **UDP**：视频直播、语音通话、DNS 查询、游戏实时对战

- 请描述 TCP 的三次握手和四次挥手过程，以及为什么需要四次挥手。
  **答案：**
  - **三次握手**（建立连接）：
    1. Client → Server：SYN=1, seq=x（请求连接）
    2. Server → Client：SYN=1, ACK=1, seq=y, ack=x+1（同意连接）
    3. Client → Server：ACK=1, seq=x+1, ack=y+1（确认）
    
    **目的**：确认双方收发能力正常，同步初始序列号
  
  - **四次挥手**（断开连接）：
    1. Client → Server：FIN=1（Client 请求关闭）
    2. Server → Client：ACK=1（确认收到 FIN）
    3. Server → Client：FIN=1（Server 也请求关闭）
    4. Client → Server：ACK=1（确认）
    
  - **为什么需要四次挥手**：
    - TCP 是全双工通信，关闭时需要分别关闭两个方向
    - 当一方发送 FIN 时，只表示不再发送数据，但仍可接收数据
    - 另一方可能还有数据要发送，所以先 ACK 确认，等数据发完再 FIN
    - 因此第 2 步和第 3 步不能合并

- HTTPS 的工作原理是什么？它是如何保证数据传输安全的？
  **答案：**
  - **工作原理**：
    1. Client 发起 HTTPS 请求
    2. Server 返回 SSL 证书（包含公钥）
    3. Client 验证证书合法性（CA 机构签发）
    4. Client 生成随机数（预主密钥），用公钥加密发送给 Server
    5. Server 用私钥解密得到预主密钥
    6. 双方根据预主密钥生成会话密钥
    7. 后续通信使用对称加密（AES 等）
  
  - **安全保证**：
    1. **机密性**：对称加密 + 非对称加密
    2. **完整性**：消息摘要算法（SHA、MD5）
    3. **身份认证**：数字证书 + CA 机构
    4. **防篡改**：数字签名

### 8.2 操作系统
- 请解释进程和线程的区别，以及它们之间的关系。
  **答案：**
  | 特性 | 进程 | 线程 |
  |------|------|------|
  | 定义 | 资源分配的基本单位 | CPU 调度的基本单位 |
  | 内存空间 | 独立地址空间 | 共享进程内存 |
  | 通信方式 | IPC（管道、消息队列、共享内存） | 直接读写数据 |
  | 切换开销 | 大 | 小 |
  | 创建/销毁 | 开销大 | 开销小 |
  | 并发性 | 低 | 高 |
  | 健壮性 | 一个进程崩溃不影响其他 | 一个线程崩溃整个进程挂掉 |
  
  - **关系**：
    1. 一个进程可以包含多个线程
    2. 至少有一个线程（主线程）才能执行
    3. 线程共享进程的资源（堆、方法区、文件句柄）
    4. 每个线程有独立的栈和程序计数器

- 什么是死锁？产生死锁的必要条件有哪些？如何避免死锁？
  **答案：**
  - **死锁**：多个线程互相等待对方释放资源，导致都无法继续执行
  
  - **四个必要条件**（同时满足才会死锁）：
    1. **互斥条件**：资源只能被一个线程占有
    2. **请求与保持**：线程在等待时不释放已占有的资源
    3. **不剥夺条件**：资源不能被强制剥夺
    4. **循环等待**：存在环形等待链
  
  - **避免方法**（破坏任一条件）：
    1. 破坏请求与保持：一次性申请所有资源
    2. 破坏不剥夺条件：允许抢占资源
    3. 破坏循环等待：按顺序申请资源（如按地址排序）
    4. 银行家算法：预先判断分配是否安全
  
  - **Java 中避免死锁**：
    - 固定顺序获取锁
    - 使用 tryLock 设置超时
    - 减少锁的粒度

- Linux 中常用的命令有哪些？请说明 ps、top、netstat 等命令的作用。^
  **答案：**
  - **ps**（Process Status）：查看进程状态
    ```bash
    ps aux          # 查看所有进程
    ps -ef | grep java  # 查找 Java 进程
    ```
  
  - **top**：实时显示进程资源占用
    ```bash
    top             # 动态刷新
    top -d 2        # 每 2 秒刷新
    ```
  
  - **netstat**（Network Statistics）：网络状态统计
    ```bash
    netstat -tlnp   # 查看 TCP 监听端口
    netstat -an | grep 8080  # 查找特定端口
    ```
  
  - **其他常用命令**：
    - `ls`：列出目录内容
    - `cd`：切换目录
    - `pwd`：显示当前路径
    - `mkdir`：创建目录
    - `rm`：删除文件
    - `cp`：复制
    - `mv`：移动/重命名
    - `cat`：查看文件内容
    - `tail -f`：实时查看日志
    - `grep`：文本搜索
    - `chmod`：修改权限
    - `kill`：终止进程
    - `df -h`：查看磁盘空间
    - `free -m`：查看内存使用
